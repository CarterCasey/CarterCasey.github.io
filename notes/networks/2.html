Internet Protocol Stack

Application

HTTP, FTP

(More detail in later classes)

Transport

TCP
End-to-end protocol, rather than just neighboring nodes

Services:

  • Give reliability in message transfer
  • In order delivery
  • Flow Control (Client end)
  • Congestion Control (Network end)

How it works:

Handshake (SYN, SYN-ACK, ACK) makes sure the message can go through

Network

IP
Narrow waist of the stack.

Services:

  • Best effort connection
  • Sends using IP address

Routes on destination (rather than content)

Link

Ethernet, wifi

Services:

  • Best effort connection
  • Sends using MAC

ARP translates MAC to IP.

Physical

Really just the wires.
Transfers bits.

Sending messages over network

  1. Start with application layer, produce message.
  2. Transport layer: add tcp header.
  3. Network layer: add ip header.
  4. Link layer: add __ header.
  5. Send over physical layer.
  6. While not at destination, repeat 3 - 5.
  7. At destination, deal with tcp and app, and if necessary send return message.

LAN can exist before getting to the network.

Network Types

Package Switched

Allows queuing, and consequently may involve queuing delay.

Circuit Switched

No queuing, only send message if there are resources available.

Causes of Delay

  • Queuing Delay: limited resources may be allocated for too many packets.
  • Propagation Delay: how far apart two nodes are physically, and speed of light.
  • Processing Delay: must process message along the way, and at the ends.
  • Transmission Delay: only so many bits may be sent at once, bandwidth of channel.

Throughput is limited to bandwidth of slowest (bottleneck) link in full path.
