TCP, cont'd

Reliability, cont'd

  • Sequence Number: byte number of packet in stream.
  • ACK Number: byte number that comes after packet (seq no + packet size).
  • If a packet is lost, send the ack for the most recent packet three times.
  • Three is considered enough for a resend, sent in response to further packets sent.

Flow Control

  • Avoid overwhelming the receiver.
  • Window Size: min of receiver window and congestion window.
  • Receiver Window: How many bytes you're willing to buffer.
    • This takes into account how many bytes have been buffered.
    • Which leads to the problem of receiver window 0.
    • Sender keeps sending small packets, receiver keeps sending ACKS with same number as sequence number of small packet and receiver window of 0.

Congestion Control

  • Don't overwhelm the network.
  • Lots of input, or input lines, to a switch with small output, causes congestion.
  • Focusing a lot of output to one port can cause congestion.
  • To avoid this, we need admission control.
  • To deal with this when it happens:
    • Queuing. Start buffering incoming packets.
    • Packet drop tells TCP that there's congestion.
  • Slow Start: Send one segment, get ACK. Send two segments, get two ACKS. Grow exponentially.
  • If we have packet drop at any point in slow start, go back to one segment and start over.
  • Slow Start Threshold: 1/2 congestion window where loss was experienced.
  • After threshold, enter Congestion Avoidance: Increase by 1 segment each time.
