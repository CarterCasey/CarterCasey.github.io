HTTP Proxies

  • Go-between for client and server.
  • Cache pages for use by other clients.
  • Benefits:
    • Client may be able to get page faster.
    • Server may have load distributed to proxy.
    • LAN may have less ingoing/outgoing traffic.

How do we know the cached page is up-to-date?

  • Just clear the cache every couple minutes.
  • Send a small check-in request every time there's a request from the client.
  • Have the server give a time after which the proxy should check in.
  • Server could send a message to update to all clients.

Actual Solutions

  • Conditional GET: if data has been modified, send it.
  • Server can say not to cache, or set an expiration time.
  • Private caching: server only allows the cache to serve specific content to one client.

Cookies

  • Keep small pieces of information about client in server.
  • Keep the actual cookie on client side. Send it to server when making get request.

Security

  • Sent in plain-text.
  • Third-party cookies.

Other Aspects of HTTP

  • You can specify content range in GET request.
  • Referral field in GET request (what website sent me here).

DNS

System to correlate domain (host) names with IP addresses.

Name Servers

Can't use one server:

  • Load will be too high.
  • Too much to store in memory.
  • Has to be available around the world (or even just relatively close).

Distributed System

  1. Start at local host.
  2. Go to local DNS server.
  3. Find closest root server (address of this bootstrapped into DNS server).
  4. Move on to TLD name server.
  5. Finally get to authoratative server of the domain name, which returns IP addr.

Lots of caching here.

Records

Have a couple fields:

  • Name: Often host name, but may vary based on type.
  • Value: A - ip addr, NS - auth server location.
  • Type: A - maps host name to ip address (auth server), NS - hostname to auth server (TLD server).
  • TTL: expiration of record.

Uses UDP because there's no handshake, doesn't need to be reliable.

Security

  • Root is the biggest target.
  • Caching alleviates some concerns.
