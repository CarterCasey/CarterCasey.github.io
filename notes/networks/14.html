Networks - Lecture 14

Mar 12

Link Layer

  • Deliver (IP) datagrams to next-hop router.

  • At link layer, packets are called frames.

  • Services:

    • Framing (sending, interpreting frames)
    • Reliability (or lack thereof?)
    • Medium Access Control (MAC)

Reliability

  • End-to-end reliability is great for correctness, but doesn't help performance.

  • For performance, however, we need reliability at a lower level.

  • ARQ (Automatic Repeat reQuest):

    • Wait for acks to know something has worked.
    • Requires timeouts, which makes it slow.

  • FEC (Forward Error Correction):

    • Add extra information to help make packet redundant.
    • For example, if we send two bytes, add an extra byte that's an xor of the two.

    • More overhead, but faster - we can correct errors.

    • Doesn't need to be confined to frame, can be on multiple frames.

Medium Access Control

  • How do we allow all connected devices to work on a shared medium?

  • Taking turns:

    • Requires an arbiter.
    • People make known their intent to send frames.
  • Channel Partitioning Protocols:
    • Allocate time.
    • Determine how frequently people.
    • Could waste resources. What if only one person wants to talk, but they have to wait for their turn while no one is talking.

  • Random Access Protocols:

    • Everyone starts sending at full rate.
    • Problem: there could be collisions.
      • Try to detect collisions.
      • Cause random backoffs.

  • Collisions:

    • Vulnerability is twice the time it takes to send frame.
    • Aloha, then slotted Aloha.
    • Slotted Aloha forces people to send in certain slots.

  • Ethernet - CSMA/CD:

    • Carrier Sense - check if medium is being used.
    • Multiple Access - multiple devices connect to medium.
    • Collision Detection - sense a change in the wire when sending.
