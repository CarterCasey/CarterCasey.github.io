IP

Routing - IP addresses

IPv4

  • 32 bit number, split into four 8 bit values.
  • First x bits is used for network part.
  • Last 32 - x bits used for host part.

  • The x is determined by class of addresses:

    • Class A: First 8 bits for network.
    • Class B: First 16 bits.
    • Class C: First 24 bits.

  • Classless InterDomain Routing (CIDR): any number of bits for network.

  • Writing out usage, we use something like 130.64.23.141/24.

  • Subnetworks can take an extra couple bits for their own net.

    • For instance, Tufts CS could get 130.64.23.141/26.

  • To get the right subnet, IP uses longest prefix matching.

Private Addresses

  • 10.0.0.0/8
  • 192.168.0.0/16

  • 172.16.0.0/16

  • Use Network Address Translation (NAT):

    • Pick a port to translate global to local IP.

Forwarding

  • We know prefix. How do we get next hop?

IP Header

  • Source and destination addresses
  • Time-to-live:
    • Mitigate loops.
    • Router decrements the field.
    • Identifier (fragment offset).
    • Flags

IPv6

  • 128 bit number for address.
  • Doesn't support fragmentation.
    • Security concerns - fragmentation forces routers to do too much.
  • Routing between v4 and v6 happens by encapsulating v6 in v4.
