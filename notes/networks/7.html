Feb 12

Review of P2P

  • Main upside is that it's more scalable. You can use the bandwidth of all the peers.
  • Not as reliable as using a dedicated server.
  • "Tit-for-tat" mechanism, selecting peers that are best for you, and then they may begin using your machine.
  • Searching uses distributed hash, so you know which nodes are responsible for certain pieces of file.

Difference from Client-Server Software

  • The machine is both server and client.
  • It's harder to discover hosts.

Transport Layer

  • IP is best effort, which means unreliable and out of order.
  • TCP offers:
    • Reliability
    • Ordering
    • Data Integrity
    • Multiplexing
  • UDP offers:
    • Data Integrity
    • Multiplexing

UDP Server

  • No listen, connect, accept
  • Get a socket, bind it, and then call recievefrom
  • recievefrom reads, but also gets client return info
  • To send back, since we have no connection, use sendto
  • Don't need a new socket, just use the same one.

De-multiplexing

  • UDP, just use destination IP and port.
  • TCP, we use multiple sockets on one port, so we have to look at the source IP/port as well.
  • First connection, hand shake, goes through default listening port.

Checksum

  • Take 16-bit checksum of message, put in header.
  • Actually take one's compliment of checksum.
