Peer-to-Peer Networks

  • File Transfer
  • Video Streaming
  • VoIP (Video over IP)

Scalability

  • For serving large networks simultaneously, P2P scales better than client/server.
  • A server sending a file out to multiple users will take as long as the slowest link (or its own upload time for all the files).
    • Time is max(NF/Us, F/dmin)
  • On P2P we don't need to upload many files from server.
    • Time is max(F/Us, F/dmin, NF/(Us + sum(U1, U_2, ...)))
  • Client/server ability to serve all members of network increases with number of members.
  • P2P levels off when more and more members are added.

BitTorrent

  • When starting, you need IP addresses of peers.
  • Large file is distributed amongst peers.
  • Downloads:
    • Send rarest chunk first, so that it can be sent to more people.
    • Prefer sending file chunk to high upload bandwidth first.
    • Add one random peer to download list. Want more people on network.
  • Uploads:
    • Choose top four of bandwidth power.
    • Also choose one random person.
  • Searching for files:
    • Could try a centralized request.
    • Try distributed approach - send request to peer, who send it to other peer.
    • Instead of those, keep Distributed Hash Table mapping content to users.
