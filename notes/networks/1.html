Networks - Lecture 1

Jan 15

Networks rely on information transfer. We must transmit robust, correct information.

  • How do we convey information between two people?
    • Convey as much information as possible at once.
    • Do it as quickly as possible.
      • Here "as possible" limits us to methods that aren't error prone.
  • How do we convey information from one person, through a group of people, to an arbitrary person in that group?
    • We may have a known structure for the group. If that's the case, we can map out which path we want to use at the beginning.
    • If we don't know the structure, we may want to have a dynamic way of finding where to send the message next. This takes a lot of overhead
  • How do we make that process secure and reliable?
    • Forward error correction: add redundant information to messages.
    • Automatic Repeat reQuest: if something was lost, re-send message.

Network protocols must be known for all parties involved.
Internet is complicated, and very large. Breaking internet into modules makes more sense.

Modules used by Internet

  1. Physical Layer: transfer of bits between entities.
  2. Link Layer: Connection with neighbor, such as ethernet and wifi.
  3. Network Layer: Deals with routing messages, IP.
  4. Transport Layer: End-to-end version of link layer, TCP.
  5. Applicaion Layer: The things that use the internet, HTTP.
