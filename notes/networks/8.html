Reliable Protocol on Unreliable Channel

Tools

Acknowledgments

  • ACK - Use when number of corrupted packets are small.
  • NACK - Use when lots of corrupted packets possible.

Retransmissions

  • Use timeouts to prevent blocking, but allow room for error.
  • Retransmit after timeout, in case packet got lost.

Sequencing

  • Sequence number of packets helps with choppy, out of order delivery.

Protocols

Stop & Wait

  • Send one packet at a time, wait for acknowledgment.
  • If ACK "never" comes, resend corresponding packet.
  • "Never," in this case, means by timeout. We don't want to wait forever.
  • If duplicate ACK is sent, just ignore second one.
  • Base timeout on propogation delay (based on previous exchanges).

Go Back N (Pipelined)

  • Know we can send N un-acknowledged packets.
  • ACK includes specific sequence number in window.
  • ACK is accumulative, so all packets up to sequence number must be fine.
  • When ACK is received, client moves up window of N packets.
  • What's the ratio between window size and sequence range.

Selective Repeat

  • Has windows like above, but get acks for all packets.
  • Timeout on each packet, and move window when all packets have arrived.
  • Receiver also moves window for all packets received.
  • Make sure sequence number size is twice as large.
