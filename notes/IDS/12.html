Models of Distributed Computing - Slides

Review of Last Class

  • Request/Response is a traffic pattern.
  • Client/Server describes the role of the end-points.
    • N-tier: More layers of servers (one ultimate server)
  • Peer-to-Peer is a different method of distributing roles.

Remote Procedure Calls

  • Make a normal function call, with little overhead, that runs remotely.
  • In C, we use proxies and stubs to "trick" C into calling the function normally.

Pros/Cons

  • Pros:

    • Transparent, easy to understand.
    • Very simple, makes using it easy.

  • Cons:

    • Languages need to support this.
    • Doesn't make sense for all problems.
    • Client and server need to agree very closely. So versioning can be an issue.

RPC Assignment

  • We can have side effects.
  • We should deal with endianness.
  • Is binary actually better than text? Think about JSON. But then, think about implementing JSON.

Distributed Object Model

  • How do we get class implementations out to the remote callee?
  • Object Orientation is highly supported by Dist OM.

Special Purpose

  • Remote File Systems, like NFS.
  • Remote Databases, device control.

Other Models

  • Broadcast, multicast (netflix might work better if we multicast)
  • Publish/Subscribe
  • Reliable Queuing - deal with partial network failure or inactivity.
