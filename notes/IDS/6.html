Introduction to TCP/IP - Slides

Internet Protocol (cont'd)

IPv6

  • IPv4 gives us 2^32 addresses, but in reality blocking makes this number much smaller.
  • IPv6 gives us 2^128, which is a hell of a lot.
  • We didn't go this far originally because 4 billion sounded like a lot, given that there were only 14 nodes in the entire network.

Domain Name System (DNS)

  • Most common case is getting ip address from host name.
  • DNS is a UDP service.
  • Other information is stored through DNS.
  • Resolution of DNS lookup is hierarchical.
    • DNS Recurser starts, looks for root server.
    • Then root server responds with TLD name server location.
    • Then gives a more specific name server for more specific domain.

IDNA: Internationalized Domain Names

  • Technically DNS is only in ASCII.
  • Someone came up with PunyCode that maps unicode to ascii in a very weird way.
  • Spoofing issues: there are unicode characters that look exactly like ascii, so when they're put in domains you can be sent somewhere completely unexpected.

Transport Layer

User Datagram Protocol

  • User-level unreliable transport of datagrams.
  • Messages may be dropped, come in out of order.
  • Program-to-program instead of host-to-host (IP). The ends are different.
  • Content is checksummed, so it's clean on arrival.
  • Try to keep to 512 bytes. Tend not to fragment.

Addressing TCP and UDP

  • Addressed using IP address/port pair.
  • Low-value ports are reserved and known.
  • Higher level ports are used for off-road programs, responses.

Why UDP?

  • Not a lot of overhead for packets.
  • Very efficient for very small idempotent messages.
  • Idempotent: Same result occurs no matter times it happens.
    • Retrieving a value, setting something to absolute value (x = 10).
    • Not setting relative values (x += 10).

Why not UDP?

  • Trying to set up your own protocol over UDP is very hard, and mistake-prone.
  • Flow control is hard, when TCP already does it for you.
  • TCP gives reliability, which is often necessary.

Transport Control Protocol

  • Stream based protocol.
  • Reliable, delivers packets in order.
