Metcalfe's Law - Slides

Note: Metcalfe's Law = Network Effects

  • Metcalfe (who co-invented the ethernet) said that the value of a network is in the number of connections there are (not just the number of nodes, though this is important).
  • So one large network is better than many small/medium sized networks.
  • Disconnected phone systems in Europe and the US are less useful than both connected.

Metcalfe's Law and the Web - Page

  • Initially, it's impressive that everyone can see a page you've just published.
  • But more impressive, you can link to other pages and let your page give that much more information. Others can also use your page to make theirs more interesting (valuable).

Implications of Metcalfe's Law - Page

If we want to extend the Web to every possible network, it has to handle a lot.

  • Accessible by whatever hardware and software you want to use.
  • Accessible wherever there is a computer.
  • Adaptable to whatever languages there are.
  • A lack of universality in the web causes divisions.
  • Business benefits from the web, but not when it tries to control it.
  • etc.

Scaling the Web - Page

  • Must uniquely identify all resources available in the world (trillions of them).
  • Have to get around all the boundaries in the world (physical, digital, legal).
  • A good Web is one that is easy to use. Memorable URIs, improvised browsing.

Pillars of the Architecture - Page

Identification: URIs

  • The scheme dictates the protocol with which to make a connection.
  • The host refers to which machine (server) we try to reach.
  • The path refers to where the resource that we want is on the server.

Interaction: HTTP

  • Send packets in a way that's understood universally by all servers on the web.

Data formats: HTML, jpg, pdf, gif, etc.

  • The web, and namely http, can handle any file format. The client is responsible for understanding the file it gets back (usually handled by browsers).

Open Standards and Open Source - Page

Open Standards: How to write something.
Open Source: What was written.

  • Open standards allow people to write the technology however they want. It lets web servers be (ideally) a black box.
  • If we only use open source software, not everyone can use it. On the other hand, open software does let people access the web freely.
