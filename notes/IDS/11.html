Models of Distributed Computing - Slides

Communicating Systems

  • We have two machines running asyncrhonously, passing messages.
  • How do we get them to work together?
  • Communicating Sequential Process (CSP) Model is the basis

Message Passing

  • We can agree to keep messages the same format, rather than just byte streams.
  • How do we synchronize? Timeouts? Retries?

Interaction Patterns

  • Request/Response: the most classic. Send a message, get a confirmation/response.
  • Client/Server: determines the role of the nodes in the system.
    • Simple, and well understood.
    • But how do we use more than two machines?
    • How do we know it works?

  • N-Tier: Multi-level Client/Server architecture.

    • Layered, each level provides service for next.
    • Useful for management, scalability, and information hiding.
    • Mid-level can cache information, serve as go-between.

  • First major n-tier system: Airline Reservations

    • Made a high request/response OS, the later versions still being used.

  • Web is usually a two or three tier system.

    • Often the middle tier is a proxy, if it exists.

Remote Procedure Call

  • Main Idea: An ordinary function call executes remotely.
