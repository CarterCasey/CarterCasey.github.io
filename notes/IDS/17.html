IDS - Lecture 17

Apr 2

(Early) Web Security - Guest Lecture by Mary Ellen Zurko

Mind the Gap - Authentication

  • Gap between human factors and security.
  • Secret management - not turtles all the way down.

Defense in depth

  • Cover all your bases.
  • Never lie to your customers.

Basic Authentication

  • Server prompts, client responds with plain-text (encoded) passwords.
  • Every server has a different authentication.
  • Who's asking for your password? Can we authenticate the authenticator?

Mind the Gap - Encryption

  • Gives you confidentiality (only you can read it).
  • Authentication
  • Integrity (Tamper Detection)
  • Replay detection (stops duplicates)

Digest Authentication

  • Hash the password, send it hashed.

  • Other side can just compare the hash.

  • Issues:

    • Clients and servers need to support each other's protocols, and the Web had already been using Basic Authentication. Can spoof for man in the middle.
    • People didn't understand why you need it, don't want to bother.

S-HTTP

  • Encrypt the HTML page on the way over.
  • Flexible - was supposed to support all encryption types.

  • Client/server can negotiate enhancements they can use.

  • The scale is crazy: users have to understand all the encryptions.

  • Flexibility introduces far too many issues.

HTTPS

  • Protocol that wraps HTTP.

  • Uses SSL/TLS to deal with encryption and authentication.

  • Forces us to trust a lot of third party authenticators.

  • If they don't use one of the many (214) authenticators, the user is faced with a weird error.

Extended Validation Certificates

  • The green padlock.
  • Doesn't mean very much when not everyone uses or cares about it.

DHTML

  • Code and Data started mixing. Bad news, because the page can suddenly start attacking you.
  • Whitelisting and Blacklisting features of HTML.

Mobile Applications

  • People still don't pay very much attention.
