Building the web from RPC

  • Why is this bad?
    • No uniform interface. Function calls are very specific.
    • The web does the same thing for all resources.

Building the Web With HTTP

  • Headers: give meta-data about http request, response.
  • Methods: GET, POST, DELETE, etc.

Technologies Used by HTTP

  • Media types, first established by email.
  • Transfer encodings.

Discoverable Web

  • You're allowed to follow whatever link you want. No matter whether it exists.
  • You don't need to know anything about the resource behind URI to attempt access.
  • Self-describing: Once you get a response, you should then understand what the resource is.
  • No side effects (to GET).

Self-Describing Web

  • When you get a resource in a response, the response should describe it.
  • Allows you to "follow your nose."

State in Protocols (Stateful vs Stateless)

  • Stateless - it doesn't matter what events happened before or after.
  • Stateful - you have to keep track of the events that happened.
  • It's much easier to scale a stateless protocol.

  • Individual requests can be simpler in stateful protocols.

  • HTTP is largely stateless, despite running over TCP.
