Modularity and Separation of Concerns Slides

Why Modularity?

  • Lets us reason better about pieces of the program.
  • Pieces of the program are well-defined.

Note: Balance
* Speed of CPU should match the size of the memory, to some extent.

OS Layering

  • Most software doesn't interact directly with the hard drive space.
  • We hide the implementation of hard drives using a raw device driver.
  • The abstraction can be different, but it's consitent throughout the operating system.

In Unix

  • File system reads from block device driver, which serves as interface for cache, which uses raw device drivers, which interacts with hard disk.
  • The layering should be clean. We can just drop the cache, still use driver.

Network Layering

  • The TCP/IP stack is mostly entirely separated.
  • Think about wi-fi. You can just switch out dial-up or ethernet for 802.11.
  • Even at HTTP layer, we have status codes and data types that evolve separately.

Noah's Theory of Simplification Choke Points

  • There's a common pattern: Complex actions hidden by a simple interface.

Layering Helps and Hurts Performance

  • Abstracting the hard drive can hide slow error corrections made in the disk.
