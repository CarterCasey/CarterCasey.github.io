KISS

Powerful tool for designing systems.

What doesn't follow this rule?

  • Unsubscribing from emails.
  • Overly complicated UIs (sis).
  • The ö in unicode: can represent as single character or two (o and ¨).

HTML & CSS

Goals of HTML (why this language?)

  • Ubiquity - as recommended by Metcalfe's Law, everyone should be using it.
  • Usable for a lot of content.
  • Should be useful to web publishing.
  • Usable with many languages (with unicode).
  • Should be able to render differently on different devices, on different screens.
  • Extensible in general.
  • Forward compatible.
  • Let it be a framework for web applications.

Format of HTML

  • In IDS, start with tag.
  • Tree structure.
  • ids allow us to make anchors (e.g. #section3).
  • Embedded elements in text (mixed content) add some structural complexity.

Links

  • Links extend beyond one domain.
  • Any element/content may be a link. Composition is simple and powerful.

CSS

  • Used to style elements in HTML.
  • Manipulates colors, margins, fonts, even structuring.
  • Selectors denote which elements will be styled.
  • Using classes lets us add semantic value to stylings.
  • You may select but id, but only one element will have id.
  • Span allows characteristics to apply to pieces of text.

Note: The separation of HTML and CSS is an example of Model/View separation.
