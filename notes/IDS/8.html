Naming cont'd

Computing Examples

  • Unix file names:
    • All chars can be used except for /, needed for paths.
    • Can have multiple names for same file.

Internet-Scale Systems

  • Don't need very big names, even though Web is very large.
  • Name design tells us how it can change, what it can include, and what it does.
  • At a high level, the Web is just a series of links to names (and the resources the refer to).

Uniform Naming

  • Uniform Naming allows us to work with things we've never seen before.
  • We don't need prior knowledge or permission to follow a link, all you need is HTTP GET.
  • Unix File System:
    • Single filename space.
    • Same space used for files, sockets, devices, pipes.
    • Names refer to file descriptors under the hood.
  • On the web, global names give consistency.
    • The same name refers to the same object.
    • Absolute global names resolve consistently to the same thing.
  • What is the "same object?" (We allow human reasoning here.)
    • A news website is the same on Tuesday and Thursday, even if the stories are all different.
    • A website can return different versions of a page depending on the language.
    • Mobile versions of sites.

Names and Addresses

Name: Identifier for something.
Address: Locator for something.

  • Some names are also addresses, but others aren't.
  • If a name doesn't let you find something, you have to convert the name into an address to find it.

Aliasing

  • Useful:
    • Absolute vs relative names.
    • Different paths to the same thing.
  • Can be confusing, hard to work with:
    • Web cache might not recognize as easily.
    • Hard to optimize with aliased pointers.

Lack of names

  • It's good to give distinct objects different names.
  • Short vs Unique names:
    • Short names are easy to remember, and very powerful.
    • They're not usually unique.
