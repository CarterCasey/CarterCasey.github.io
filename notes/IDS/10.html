Request for Comments (RFC) - Slides

  • IETF: Composed of individuals.
  • Open Standards: A means for everyone to interact on the web/internet/etc, without forcing them to use the same software.
  • Open Source: Still useful for people to interact with web without dealing with propriety.

Short History of RFC

  • Steve Crocker wanted to have an informal way of laying down notes from IETF.
  • Didn't want to seem presumptuous, so called it Request for Comments.
  • Remain, to this day, the documentation for Internet Protocols.

URIs and RFC 3986 - Slides

Names in HTTP

  • Schemes hold names. They (http, etc) refer to a space of names.
  • The host is a name, and the path is a different kind of name.
  • Even the status code and content-type is a name.
    • How do decide what the content-type means.

URI breakdown

  • Scheme: Name different kinds of things, which are accessed in different ways.
  • Authority: The host, the one responsible for this name.
  • // Tim doesn't think the second slash was necessary.
  • Path: The actual name of the resource.
  • Query: Often, all queries are processed by the same script. Text left of ? is used to pick an agent to process query.
  • # fragment ...

Characteristics of URIs

  • URI references can be absolute and relative
  • Considered an address, because you can use URI to find a resource.
  • Mostly global.
  • Are literally the structure of the web. Reveal the hierarchy of underlying systems.
  • Environmental constraints: ASCII-only because of DNS (and such), % encoding for non-readable or banned characters.
  • HTTP redirection allows indirect identification (like looking up Noah).

Grammars

  • Powerful tool for parsing and checking names/specs.
  • They're formal, and cannot be debated.
